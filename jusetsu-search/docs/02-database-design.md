## 2.2 エンティティ一覧

| エンティティ | 役割 |
|---|---|
| `makers` | メーカー（概要・公式サイトURL・ロゴ） |
| `categories` | 商品カテゴリ（キッチン等11種、ツリー可） |
| `products` | 商品（メーカー×カテゴリに属する中心エンティティ） |
| `product_images` | 商品画像（複数・表示順） |
| `product_assets` | 商品資料（カタログ/仕様書/CAD/取説のファイル参照） |
| `product_specs` | 商品仕様（キー・バリュー型の柔軟な仕様情報） |
| `tags` | 特徴タグ（ペニンシュラ、タンクレス等） |
| `category_tags` | カテゴリと利用可能タグの紐付け（絞り込みUI生成用） |
| `product_tags` | 商品とタグの中間テーブル |
| `cases` | 施工事例（商品に紐付く） |
| `case_images` | 施工事例の画像 |
| `users` | 一般ユーザー（お気に入り保存用、任意登録） |
| `favorites` | お気に入り（ユーザー or 匿名ID × 商品） |
| `admin_users` | 管理者アカウント |

## 2.3 主要な関連（カーディナリティ）

- `makers (1) — (N) products`
- `categories (1) — (N) products`
- `products (1) — (N) product_images`
- `products (1) — (N) product_assets`
- `products (1) — (N) product_specs`
- `products (N) — (N) tags`（`product_tags` 経由）
- `categories (N) — (N) tags`（`category_tags` 経由：カテゴリ別の絞り込み候補）
- `products (1) — (N) cases`
- `cases (1) — (N) case_images`
- `users (1) — (N) favorites`、`products (1) — (N) favorites`

## 2.4 設計上のポイント

1. **仕様情報は `product_specs` でキー・バリュー化**
   カテゴリごとに項目が大きく異なる（キッチンの天板素材 vs 給湯のタンク容量）ため、
   固定カラムではなく EAV 的に持ち、比較画面で動的にマトリクス化する。
   主要比較項目（価格・サイズ・カラー）は `products` 本体にも代表値を持たせ高速表示。

2. **タグは「カテゴリ別」に提示**
   `category_tags` でカテゴリごとに使えるタグ集合を定義し、絞り込みUIを自動生成。
   （例：キッチン選択時のみ「アイランド」「セラミック天板」を表示）

3. **資料は種別enumで管理**
   `product_assets.asset_type`：`catalog` / `spec_sheet` / `cad` / `manual`。
   ファイルは外部ストレージ、DBにはURL・ファイル名・サイズ・拡張子を保持。

4. **お気に入りは匿名対応**
   `favorites` は `user_id`（NULL可）と `anon_id`（ブラウザ生成UUID）の両対応。
   後からアカウント登録した際に `anon_id` を `user_id` へマージ可能。

5. **価格は「価格帯」絞り込みに対応**
   `products.price`（税抜・参考定価, NULL許容＝オープン価格）と
   `price_band`（自動算出 or 手動：`~10万` / `10-30万` 等）で絞り込み高速化。
