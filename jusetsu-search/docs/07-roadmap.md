# 7. 開発ロードマップ

「まず施工例から逆引きし、検索して比較できる」最小価値を最短で立ち上げ、段階的に拡張する。
建材サーチの住設版として、**データ網羅性（各社カタログの取り込み）**と**メーカー入稿＋承認**を中核に据える。

## 全体方針

- **Phase 0** で基盤（リポジトリ/DB/認証/CI/ストレージ）を整える
- **Phase 1（MVP）** で「検索（サジェスト付）→ 詳細 → 資料DL」と「施工例 → 逆引き」を公開
- **Phase 2** で「絞り込み・比較・お気に入り（登録不要）」と**カタログ一括取り込み**を追加
- **Phase 3** で「メーカー入稿＋運営承認」「施工例の充実」「性能・SEO」
- **Phase 4** で「ユーザー機能・送客・分析・取り込み自動化」

---

## Phase 0：基盤構築（1〜2週）

- モノレポ構成（FE: Next.js / API: NestExpress / DB: PostgreSQL / Storage: S3互換）
- DBスキーマ・マイグレーション（[テーブル設計](./05-table-design.md)）
- 認証基盤（運営/メーカー/一般のJWT）、署名付きURLアップロード
- CI/CD、Lint/Format、環境変数管理、デザイントークン・UI雛形

**完了条件**：ステージングで空アプリが動き、マイグレーションが通る。

## Phase 1：MVP（2〜4週）

- データ投入用の**管理画面（最小）**：メーカー・カテゴリ・商品・資料・画像・施工例の登録/編集
- **トップ**（検索バー＋人気/新着＋施工例）
- **商品一覧/詳細**（画像・説明・仕様・資料DL・商品URL、価格は `price_kind` で出し分け）
- **メーカー一覧/詳細**
- **施工例一覧/詳細（逆引き：使用設備一覧 → 商品詳細）** ← 建材サーチの肝を最初から
- **横断検索＋入力サジェスト**（`/search`, `/search/suggest`）

**完了条件**：実データで「施工例 → 設備 → 商品詳細 → カタログ/CAD/仕様書DL」「検索 → 詳細」が成立。スマホOK。

## Phase 2：比較・絞り込み・お気に入り・カタログ取り込み（3〜4週）

- **絞り込み**（カテゴリ・メーカー・価格帯・特徴タグ／カテゴリ別タグの動的表示・facet件数）
- **比較**（最大4・比較トレイ＋比較ページ・仕様マトリクス・登録不要）
- **お気に入り**（匿名 `anon_id` → 任意ログインでマージ）／並び替え
- **★各社Webカタログの一括取り込み（インジェスト）基盤**（[取り込み設計](./08-catalog-ingestion.md)）
  - CSV/Excelテンプレート取り込み、資料(PDF/CAD)の一括アップロード、取り込みジョブ管理・重複検出

**完了条件**：営業が「絞り込み→比較→お気に入り」を数クリックで回せ、運営が各社データを一括投入できる。

## Phase 3：メーカー入稿＋承認・施工例充実・性能・SEO（3〜4週）

- **メーカーポータル**（`/maker`）：自社商品の入稿（draft → submit）
- **運営承認キュー**（`/admin/reviews`）：approve/reject＋差戻しコメント、`product_review_logs`
- 施工例の本格運用（スタイルタグ、関連事例、商品詳細の採用事例タブ）
- 検索高度化（Meilisearch/OpenSearch 移行、サジェスト精度）
- 性能（CDNキャッシュ・画像最適化・Core Web Vitals）、SEO（構造化データ・sitemap.xml・OGP）

**完了条件**：本番公開品質。メーカー入稿が回り、データが自走で増える。

## Phase 4：ユーザー機能・送客・分析・取り込み自動化（継続）

- ユーザー属性（営業/設計/工務店/施主）、比較リスト共有・PDF出力
- 「見積/問い合わせ」導線（メーカー・販売店への送客）
- レコメンド（よく一緒に比較される商品、施工例からの関連設備）
- アクセス解析の精緻化（`product_views_daily` 等を必要に応じ追加）
- **取り込みの自動化**：メーカー提供フィード（CSV/JSON/API）への対応、定期同期、差分更新

## リスクと対策

| リスク | 対策 |
|---|---|
| データ整備が重い（各社カタログ） | Phase2で**CSV一括取り込み＋資料一括アップロード**、Phase3でメーカー入稿に移行し自走化 |
| 他社サイトの自動取得は規約/著作権リスク | **許諾済みデータ・公式提供フィード・入稿を基本**とし、無断クロールは行わない（[取り込み設計](./08-catalog-ingestion.md) 参照） |
| 仕様項目がカテゴリで多様 | `product_specs` のkey/valueで吸収、比較は共通キー優先表示 |
| 大容量CAD/カタログ配信 | 署名付きURLで直アップロード/直DL、CDN配信 |
| 検索精度 | Phase1はpg_trgm、Phase3で専用検索エンジンへ移行できる抽象化 |
| 入稿品質のばらつき | 運営承認フロー＋必須項目バリデーション＋差戻しコメント |
