# 7. 開発ロードマップ

「まず検索して比較できる」最小価値を最短で立ち上げ、段階的に拡張する。

## 全体方針

- **Phase 0** で基盤（リポジトリ/DB/認証/CI）を整える
- **Phase 1（MVP）** で「検索→詳細→資料DL」までを公開
- **Phase 2** で「絞り込み・比較・お気に入り」を追加し、営業が毎日使えるUIへ
- **Phase 3** で「施工事例・管理画面の充実・パフォーマンス」
- **Phase 4** で「ユーザー機能・分析・拡張」

---

## Phase 0：基盤構築（1〜2週）

- リポジトリ/モノレポ構成（FE: Next.js / API: NestExpress / DB: PostgreSQL）
- DBスキーマ作成・マイグレーション（[テーブル設計](./05-table-design.md)）
- 認証基盤（管理者JWT）、ファイルストレージ（S3互換）接続
- CI/CD、Lint/Format、環境変数管理
- デザイントークン（余白・タイポ・カラー）とUIコンポーネント雛形

**完了条件**：ローカル/ステージングで空のアプリが動き、マイグレーションが通る。

## Phase 1：MVP（2〜4週）

- データ投入用の**管理画面（最小）**：メーカー・カテゴリ・商品・資料・画像の登録/編集
- **トップページ**（検索バー＋人気/新着カテゴリ・商品の枠）
- **商品一覧**（カテゴリ・キーワード検索）
- **商品詳細**（画像・説明・仕様・資料DL・商品URL）
- **メーカー一覧/詳細**
- **横断検索**（メーカー/商品/シリーズ）

**完了条件**：実データで「検索 → 商品詳細 → カタログ/CAD/仕様書DL」が成立。スマホ表示OK。

## Phase 2：比較・絞り込み・お気に入り（2〜3週）

- **絞り込み**（カテゴリ・メーカー・価格帯・特徴タグ／カテゴリ別タグの動的表示・facet件数）
- **比較機能**（最大4商品、比較トレイ＋比較ページ、仕様マトリクス）
- **お気に入り**（匿名IDベース→任意でログインへマージ）
- 並び替え（人気/新着/価格）

**完了条件**：営業が商談中に「絞り込み→比較→お気に入り」を2〜3クリックで回せる。

## Phase 3：施工事例・管理充実・性能（2〜3週）

- **施工事例**一覧/詳細、商品詳細への事例タブ、管理画面の事例登録
- 管理画面の本格化（一括操作、画像並び替え、下書き/公開フロー、権限）
- 検索高度化（Meilisearch/OpenSearch 導入、サジェスト）
- パフォーマンス（CDNキャッシュ、画像最適化、Core Web Vitals）
- SEO（構造化データ、サイトマップXML、OGP）

**完了条件**：本番公開品質。事例が回り、検索が高速。

## Phase 4：ユーザー機能・分析・拡張（継続）

- ユーザー登録（営業/設計/工務店/施主の属性）と、比較リスト共有・PDF出力
- 「見積/問い合わせ」導線（メーカー or 販売店への送客）
- レコメンド（よく一緒に比較される商品）
- アクセス解析・人気指標の精緻化（`product_views_daily` 集計）
- メーカー向け管理ポータル（自社商品の入稿）

## マイルストーン早見表

|---|---|---|---|
| 0 | 1〜2週 | 基盤・DB・CI | 開発開始 |
| 1 | 2〜4週 | 検索/詳細/資料DL/最小管理 | 単サイトで資料収集が完結 |
| 2 | 2〜3週 | 絞り込み/比較/お気に入り | 営業が比較検討を完結 |
| 3 | 2〜3週 | 施工事例/管理充実/性能/SEO | 本番公開 |

## リスクと対策

| リスク | 対策 |
|---|---|
| データ整備（商品・資料）が重い | Phase1で入稿しやすい管理UI＋CSV一括登録を先行 |
| 仕様項目がカテゴリで多様 | `product_specs` のkey/value設計で吸収、比較は共通キー優先表示 |
| 大容量CAD/カタログの配信 | 署名付きURLで直アップロード/直DL、CDN配信 |
| 検索精度 | Phase1はpg_trgm、Phase3で専用検索エンジンへ移行できる抽象化 |
| 著作権/掲載許諾 | メーカー資料の掲載可否管理フラグ・出典明記、公式URLへの誘導を基本 |
