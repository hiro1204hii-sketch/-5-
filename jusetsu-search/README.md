# 住設サーチ（Jusetsu Search）

住宅設備機器に特化した検索・比較サイトの**設計ドキュメント**一式です。
[建材サーチ](https://kenzai-search.jp) の住設版として、同じ操作感で並び立つことを目指します。
住宅設備の情報を一箇所に集約し、**メーカーサイトを何社も見なくても住設サーチだけで
比較検討できる**ことがゴールです。

## ターゲットユーザー

| ユーザー | 主な利用シーン |
|---|---|
| 住宅営業 | 商談中に施主へ商品提案・比較資料を即提示 |
| 住宅設計 | 仕様検討・CADデータ取得・図面落とし込み |
| 工務店 | 採用設備の選定・カタログ/仕様書の一括取得 |
| リフォーム会社 | 既存設備からの置換候補比較 |
| 施主 | 入れたい設備のイメージ収集・施工例閲覧 |

## サービスコンセプト

> 住宅設備の情報を一箇所に集約する検索サイト。
> メーカーサイトを横断せず、住設サーチだけで比較検討を完結させる。
> ECのような感覚で「探す → 見る → 比べる → 資料を取る」が止まらない。

## 対象カテゴリ

キッチン / ユニットバス / 洗面化粧台 / トイレ / 給湯設備 / 換気設備 / 空調設備 /
太陽光発電 / 蓄電池 / 宅配ボックス / 照明

## 設計の核となる原則（建材サーチに揃える4本柱）

1. **登録不要で全機能が使える**
   検索・閲覧・**資料DL（カタログ/仕様書/CAD/取説）・お気に入り・比較**まで、
   ログインなしで完結する。マイページ（任意登録）は端末間同期と比較リスト保存の“おまけ”。

2. **施工例（空間イメージ）からの逆引きを主役にする**
   施工例（`/portfolios`）を一級の入口とし、**1事例に複数の住宅設備を紐付け**。
   「この空間で使われている設備一覧 → 各商品詳細」へ最短で辿れる。

3. **メーカー入稿＋運営承認モデル**
   メーカー担当者が自社商品を入稿し、運営が承認して公開。
   情報の鮮度と網羅性をスケールさせる（権限と公開ワークフローを中核に設計）。

4. **EC感覚の検索体験**
   常設検索バー＋**入力サジェスト**、多軸の絞り込み（カテゴリ/メーカー/価格帯/特徴タグ）、
   facet件数表示。最短2クリックで比較表へ。

## 想定技術スタック（提案）

| レイヤ | 技術 |
|---|---|
| フロントエンド | Next.js (App Router) + TypeScript + Tailwind CSS |
| 状態管理 | React Query（サーバ状態）/ Zustand（お気に入り・比較） |
| バックエンド | Node.js + NestJS（or Express）/ REST API |
| データベース | PostgreSQL（全文検索＋サジェストは pg_trgm、将来 Meilisearch へ抽象化） |
| ファイル保管 | S3 互換ストレージ（カタログ/CAD/画像、署名付きURLで直アップロード/直DL） |
| 認証 | 管理/メーカー：メール+パスワード(JWT) / 一般：匿名(anon_id)＋任意登録 |
| インフラ | Vercel（FE）+ Render/Fly.io/AWS（API・DB）+ CDN |

## デザイン原則

- シンプル・見やすい・検索しやすい
- スマホ完全対応のレスポンシブ
- 余白を活かしたレイアウト
- 住宅会社の営業担当が毎日使いたくなるUI（最短2クリックで比較表へ）

## 価格表示の運用ルール

住宅設備はオープン価格・定価非公開が多いため、価格は必須としない。
- 価格あり … メーカー希望小売価格（税抜）を表示し、価格帯絞り込みの対象に含める
- 価格なし … 「オープン価格」または「要問合せ」と明示し、価格帯絞り込みからは除外
